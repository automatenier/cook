# New Client Folder Generator

> When a new client signs, run this to create all necessary folders across Cook (OneDrive-synced).
> VA can follow this manually, or Jordan can script it.

---

## Quick Command (Bash)

Replace `CLIENT_NAME` with the actual client name (lowercase, no spaces, use hyphens).

```bash
CLIENT="client-name"

# Content production folders
mkdir -p "content/clients/${CLIENT}/a_roll"
mkdir -p "content/clients/${CLIENT}/footage"
mkdir -p "content/clients/${CLIENT}/assets"
mkdir -p "content/clients/${CLIENT}/projects"

# Fulfillment deliverables folder
mkdir -p "Fulfillment/_CLIENT_DELIVERABLES/${CLIENT}"

# Copy templates
cp "content/clients/_template/client_brief.md" "content/clients/${CLIENT}/client_brief.md"
cp "content/clients/_template/footage_checklist.md" "content/clients/${CLIENT}/footage_checklist.md"
```

---

## What Gets Created

### 1. Content Production (`content/clients/[name]/`)

```
content/clients/[CLIENT]/
├── client_brief.md          ← Copied from _template, fill with client data
├── footage_checklist.md     ← Copied from _template, fill after onboarding
├── a_roll/                  ← Talking head clips (shot once, reused forever)
├── footage/                 ← B-roll, screen recordings, new clips
├── assets/                  ← NanoBanana images, overlays, thumbnails
└── projects/                ← Per-content-piece folders (YYMMDD_slug/)
```

### 2. Fulfillment Deliverables (`Fulfillment/_CLIENT_DELIVERABLES/[name]/`)

```
Fulfillment/_CLIENT_DELIVERABLES/[CLIENT]/
├── offer_sheet.md           ← Generated by offer agent
├── offer_sheet.json         ← Structured data
├── content_audit/           ← Audit report + calendar
├── keyword_cta_pairs.md     ← Documented CTA keywords
├── weekly_reports/          ← Ongoing performance reports
└── notes.md                 ← Session notes, feedback, misc
```

### 3. GDrive Shared Folder (created separately by VA)

See `va_setup_sop.md` Section 5 for full GDrive structure.

## Filename Convention Reminder

```
[CLIENT]_[ASSET-TYPE]_[DESCRIPTOR]_[SEQ].[ext]
```

| Type | Example |
|------|---------|
| `aroll` | `busyfit_aroll_cta-dm_01.mp4` |
| `broll` | `busyfit_broll_gym-training_01.mp4` |
| `screen` | `busyfit_screen_dashboard_01.mp4` |
| `nano` | `busyfit_nano_hook-busy-mom_01.png` |
| `overlay` | `busyfit_overlay_cta-free_01.png` |
| `thumb` | `busyfit_thumb_transformation_01.png` |
| `ref` | `busyfit_ref_steal-competitors_01.mp4` |

Full convention: `content/README.md`

## OneDrive Sync Note

All `Cook/` folders are synced via OneDrive. When you create folders locally:
- They auto-sync to cloud
- Team members with shared access can see them
- No manual upload needed
- Large video files may take time to sync — check OneDrive status icon
