The Cook workspace uses prefixes to make every folder's purpose instantly obvious. One guide per prefix group.

| Guide | Prefix | What it covers |
|-------|--------|---------------|
| [Guide-AI_.md](Guide-AI_.md) | `AI_` | AI_BRAIN, AI_MEMORY, AI_Tools, AI_n8n — all agent-owned systems |
| [Guide-PDCT_.md](Guide-PDCT_.md) | `PDCT_` | PDCT_JO_Consult, PDCT_Real_Estate — client acquisition CRMs |
| [Guide-HMN_.md](Guide-HMN_.md) | `HMN_` | HMN__Command, HMN_A INPUTS — human control surfaces and input capture |
| [Guide-VLT_.md](Guide-VLT_.md) | `VLT_` | VLT_Content, VLT_Content/__VLT_OBSVAULT — content production and knowledge vault |

## Quick routing rule
```
Got content to produce?     → VLT_Content/03_HMN_GUIDELINES/[client]/
Got a new lead or CRM data? → PDCT_JO_Consult/ or PDCT_Real_Estate/
Got raw input (meeting/idea)? → 02 HMN_A INPUTS/ (Capture OS)
Building or running a tool? → AI_Tools/
Need agent priming/context? → AI_BRAIN/
Checking n8n automations?   → AI_n8n/
```
